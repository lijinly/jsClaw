# 设计方案：工具执行验证机制

## 背景

当前 `SkillRegistry.js` 中的 `executeToolCalls` 函数直接执行工具调用，缺乏：
1. **前置验证**：参数校验、权限检查、依赖检查
2. **后置验证**：结果格式验证、业务规则校验
3. **执行追踪**：执行时长、资源消耗、执行链路

## 方案

在 `SkillRegistry.js` 中新增验证层，最小改动原则：

### 1. 新增验证配置结构

```javascript
// 每个 skill 可选择性地添加验证规则
const validationSchema = {
  beforeExecute: (skill, args) => { /* 返回 { valid: boolean, error?: string } */ },
  afterExecute: (skill, result) => { /* 返回 { valid: boolean, error?: string } */ },
  timeout: 30000,  // 超时毫秒
  retryable: true, // 是否可重试
};
```

### 2. 改造 executeToolCalls

```javascript
export async function executeToolCalls(toolCalls, options = {}) {
  const results = [];
  for (const call of toolCalls) {
    const skill = registry.get(call.function.name);

    // 前置验证
    if (skill?.validation?.beforeExecute) {
      const validation = skill.validation.beforeExecute(skill, args);
      if (!validation.valid) {
        results.push({ tool_call_id: call.id, content: `验证失败: ${validation.error}` });
        continue;
      }
    }

    // 执行（带超时）
    try {
      const result = await Promise.race([
        skill.execute(args),
        new Promise((_, reject) =>
          setTimeout(() => reject(new Error('执行超时')), skill.validation?.timeout || 30000)
        )
      ]);

      // 后置验证
      if (skill?.validation?.afterExecute) {
        const validation = skill.validation.afterExecute(skill, result);
        if (!validation.valid) {
          results.push({ tool_call_id: call.id, content: `结果验证失败: ${validation.error}` });
          continue;
        }
      }

      results.push({ role: 'tool', tool_call_id: call.id, content: /* ... */ });
    } catch (err) {
      results.push({ role: 'tool', tool_call_id: call.id, content: `执行出错: ${err.message}` });
    }
  }
  return results;
}
```

### 3. 注册时支持验证规则

```javascript
export function registerSkill({ name, description, parameters, execute, validation }) {
  registry.set(name, { name, description, parameters, execute, validation });
}
```

## 验证点

1. 参数校验失败时立即返回错误，不执行工具
2. 执行超时时抛出明确错误
3. 结果验证失败时返回验证错误信息
4. 原有 skill 注册方式完全向后兼容
