# 设计方案：增强上下文管理

## 背景

当前 `ContextOptimizer.js` 已实现基础的上下文裁剪，存在以下增强空间：

1. **保留策略单一**：目前只支持保留最近 N 轮，无法区分消息重要性
2. **token 估算粗糙**：使用线性估算（字符数 * 0.25），精度不足
3. **裁剪策略固定**：无法根据场景切换不同策略

## 方案

在 `ContextOptimizer` 中新增增强配置，最小改动原则：

### 1. 新增配置项

```javascript
constructor(options = {}) {
  // ... 现有配置 ...

  // 新增配置
  this.priorityKeywords = options.priorityKeywords || [];  // 高优先级关键词
  this.strategy = options.strategy || 'simple';  // 'simple' | 'aggressive' | 'smart'
}
```

### 2. 增强 prune 方法

```javascript
prune(messages) {
  if (!this.autoPrune || messages.length < 6) return messages;
  if (!this.needsPrune(messages)) return messages;

  const systemMessages = messages.filter(m => m.role === 'system');
  const nonSystemMessages = messages.filter(m => m.role !== 'system');

  // 根据策略选择裁剪方式
  switch (this.strategy) {
    case 'aggressive':
      return this._aggressivePrune(systemMessages, nonSystemMessages);
    case 'smart':
      return this._smartPrune(systemMessages, nonSystemMessages);
    default:
      return this._simplePrune(systemMessages, nonSystemMessages);
  }
}

_smartPrune(systemMessages, nonSystemMessages) {
  const recentMessages = nonSystemMessages.slice(-this.preserveRecent * 2);

  // 识别高优先级消息（含关键词）
  const priorityMessages = nonSystemMessages
    .slice(0, -this.preserveRecent * 2)
    .filter(m => this._containsPriorityKeyword(m.content));

  // 合并：系统消息 + 高优先级 + 最近消息
  return [...systemMessages, ...priorityMessages, ...recentMessages];
}

_containsPriorityKeyword(content) {
  if (!this.priorityKeywords.length) return false;
  return this.priorityKeywords.some(kw =>
    content?.toLowerCase().includes(kw.toLowerCase())
  );
}
```
