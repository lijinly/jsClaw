# 设计方案：错误处理与恢复机制

## 背景

当前 `Manager.js` 和 `Task.js` 中已有基础的重试逻辑：

- `Task.js`：`maxAttempts`、`attempts`、`canRetry()`、`fail()` 支持重试状态
- `Manager.js`：在 `_executeTask` 中调用 `task.canRetry()` 判断是否重试

存在不足：
1. **重试策略单一**：固定间隔重试，不区分错误类型
2. **缺少熔断**：持续失败时无熔断机制
3. **错误分类缺失**：网络超时、权限错误、业务错误混为一谈

## 方案

在 `Manager.js` 中新增错误处理配置和分类机制，最小改动原则：

### 1. 新增错误分类

```javascript
// 错误类型枚举
const ErrorType = {
  TRANSIENT: 'transient',      // 瞬时错误（网络超时、临时不可用）→ 可重试
  PERMANENT: 'permanent',      // 永久错误（权限不足、参数错误）→ 不重试
  UNKNOWN: 'unknown',          // 未知错误 → 有限重试
};

// 错误分类函数
function classifyError(error) {
  const msg = error.message?.toLowerCase() || '';

  if (msg.includes('timeout') || msg.includes('econnrefused') || msg.includes('enotfound')) {
    return ErrorType.TRANSIENT;
  }
  if (msg.includes('permission') || msg.includes('unauthorized') || msg.includes('invalid')) {
    return ErrorType.PERMANENT;
  }
  return ErrorType.UNKNOWN;
}
```

### 2. 增强 Manager 配置

```javascript
constructor(options) {
  // ... 现有配置 ...

  // 新增错误处理配置
  this.errorConfig = {
    maxRetries: managerConfig.maxRetries ?? 3,
    baseDelayMs: managerConfig.baseDelayMs ?? 1000,     // 基础重试间隔
    maxDelayMs: managerConfig.maxDelayMs ?? 30000,      // 最大重试间隔
    exponentialBackoff: managerConfig.exponentialBackoff ?? true,  // 指数退避
    circuitBreakerThreshold: managerConfig.circuitBreakerThreshold ?? 5,  // 熔断阈值
    ...managerConfig.errorConfig,
  };

  // 熔断状态
  this._circuitOpen = false;
  this._consecutiveFailures = 0;
}
```

### 3. 改造重试逻辑

```javascript
async _executeTask(goal, task, member) {
  try {
    const result = await this._executeToolWithTimeout(task);

    // 成功时重置熔断计数
    this._consecutiveFailures = 0;

    goal.onTaskComplete(task.id, true, result);
    return result;

  } catch (error) {
    const errorType = classifyError(error);

    // 永久错误不重试
    if (errorType === ErrorType.PERMANENT) {
      task.fail(error.message);
      goal.onTaskComplete(task.id, false, null, error.message);
      this._handleConsecutiveFailure();
      return;
    }

    // 检查熔断
    if (this._circuitOpen) {
      task.fail('服务熔断中，暂停重试');
      goal.onTaskComplete(task.id, false, null, '服务熔断');
      return;
    }

    // 计算退避延迟
    const delay = this._calculateBackoff(task.attempts);

    if (task.canRetry()) {
      console.log(`[Manager:${this.id}] ${delay}ms 后重试 Task: ${task.id}`);
      setTimeout(() => this._executeTask(goal, task, member), delay);
    } else {
      task.fail(error.message);
      goal.onTaskComplete(task.id, false, null, error.message);
      this._handleConsecutiveFailure();
    }
  }
}

_calculateBackoff(attempt) {
  const { baseDelayMs, maxDelayMs, exponentialBackoff } = this.errorConfig;
  if (exponentialBackoff) {
    return Math.min(baseDelayMs * Math.pow(2, attempt), maxDelayMs);
  }
  return baseDelayMs;
}

_handleConsecutiveFailure() {
  this._consecutiveFailures++;
  if (this._consecutiveFailures >= this.errorConfig.circuitBreakerThreshold) {
    this._circuitOpen = true;
    console.warn(`[Manager:${this.id}] 熔断开启，${this.errorConfig.maxDelayMs}ms 后自动恢复`);

    // 定时器后自动恢复
    setTimeout(() => {
      this._circuitOpen = false;
      this._consecutiveFailures = 0;
      console.log(`[Manager:${this.id}] 熔断恢复`);
    }, this.errorConfig.maxDelayMs);
  }
}
```

## 验证点

1. 网络超时错误自动重试，使用指数退避
2. 权限错误立即失败，不重试
3. 连续失败 5 次后触发熔断
4. 原有 `maxAttempts` 配置完全兼容
